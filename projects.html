Packing Problem: 2D and 3D Packingüì¶

Sep 2021 - Jan 2022

Packing problems play an essential rol in operation research. Companies can drastically safe costs by using the packing strategies generated by our algorithms as starting point.

We developed 2D packing algorithms, in the form of Pentris. Packing was tried using brute-force but later with genetic algorithm where fitness was determined based on the pentomino to pack and the 'shape' of the exisiting pentominoes.

We used genetic algorithm, greedy algoithm and dancing links to pack a 3D container. This GUI was build using JavaFX.

Developed algorithms from scratch to pack 2D or 3D packages efficiently into a container. Initially built a Tetris-style solver, gradually advancing to 3D packages. This project was an interesting workout in the domain of algorithms, especially as it serves as an excellent sandbox for learning how to work with greedy and genetic algorithms. Additionally, it provided an opportunity to implement Donald Knuth's "Dancing Links," which can also be used for the famous exact cover problem, Sudoku. This project included GUIs programmed in Swing and JavaFX.

Technologies

Java
JavaFX

Demos:

In this video we can see the brute force bot in action. It clearly does not make any informed moves leading to a quick game over state.
The genetic algorithm performs better than the first bot by optimizing fitness based on factors like the height of the board, the number of locked empty squares, and the variation in column bumpiness. It also considers how many lateral grids are filled with pentominoes and how many blocks touch the bottom of the grid, all of which improve its placement strategy. The more these features are fine-tuned, the higher the bot‚Äôs score, resulting in more efficient gameplay overall.

The project advanced by developing a JAVA-FX applicaiton with backend algorithms to pacakage 3D items. These items 'priority' was based on their weight and value, it is therefore similar to the knapsack problem.

The greedy algorithm is used here to package 'irregular' shaped boxes into containers. Including more factors with approriate weights, improves the efficiency of packing. The factors considered in 3D bin packing include total empty space, touching walls (x, y, z), number of touching parcel sides, gaps between parcels (x, y, z), and the total container value.
